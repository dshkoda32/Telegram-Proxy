# Telegram-Proxy

Простой скрипт для запуска локального прокси, чтобы обеспечить доступ к Telegram при ограничениях сети.
Скрипт устанавливает/обновляет `telemt`, инициализирует конфиг, перезапускает сервис и выводит готовую `tg://proxy` ссылку для подключения.

Внутри `telegram-proxy.sh` есть настраиваемые параметры, которые можно изменить перед запуском:
- `PORT` — порт прокси-сервера;
- `DOMAIN` — домен для TLS-маскировки;
- `VERSION` — версия `telemt` (если не указана, берется последний релиз);
- `MAX_CONNECTIONS` — лимит одновременных TCP-подключений;
- `BUF_DIVISOR` — делитель буферов для снижения потребления памяти.

Дополнительно скрипт автоматически изменяет `config.toml`, чтобы применить рабочие настройки:
- отключает `beobachten`, `IPv6`, `STUN`, телеметрию и `WEB-API`;
- настраивает таймауты клиентов (`client_keepalive`, `client_ack`);
- устанавливает тихий режим логирования (`log_level = "silent"`);
- применяет лимит подключений (`max_connections`);
- уменьшает сетевые/крипто-буферы в соответствии с `BUF_DIVISOR`.

Чтобы установить, загрузите скрипт в папку `root` и выполните:

```
chmod +x /root/telegram-proxy.sh && /root/telegram-proxy.sh
```
